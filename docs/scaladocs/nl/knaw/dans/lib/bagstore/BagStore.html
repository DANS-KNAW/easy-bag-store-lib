BagStore 

Companion object BagStore

trait BagStore extends AnyRef

Interface to a bag store. Use the constructor on the companion object to create an instance to work with.

Bag store operations

The methods of this trait implement some of the operations as defined on the Definitions page of the bag store project:

  • ADD: BagStore.add
  • GET: the various overloads of get
  • ENUM (for bags): BagStore.enum

The other operations are defined in Item and its subclasses. To retrieve an item object, use one of overloads of get passing it the appropriate item-id. Use the item object to access the properties of corresponding bag or file in the bag store, or to perform one of the operations on it:

  • ENUM (for files in a bag): BagItem.enum
  • DEACTIVATE: BagItem.deactivate
  • REACTIVATE: BagItem.reactivate
  • ERASE: RegularFileItem.erase

Utility functions

Additionally, some utility function are provided:

  • verify - to verify that a bag store is consistent
  • prune - to minimize a bag before adding it to this bag store
  • complete - to complete a minimized bag

Example code 

val bagStore = BagStore(File("/data/bagstore"), File("/data/staging"))
val bagToAdd = File("/home/jdoe/bag1")

for {
  // Replace files already in bag store with fetch-includes
  _ <- bagStore.prune(bagToAdd)
  // Double check that bagToAdd is virtually-valid (will also be done by add)
  virtValid <- bagStore.isVirtuallyValid(bagToAdd)
  // Add it to the bag store
  id <- bagStore.add(File("/home/jdoe/bag1"))
  // Check that you can retrieve it
  item <- bagStore.get(id)
  // Where is the bag stored?
  location <- item.getLocation
  // Print the location
  _ = println(location)
  // Deactivate the bag
  _ <- item.deactivate()
} yield ()

  1.  abstract def add(bag: File, optUuid: Option[UUID] = None, move: Boolean = false): Try[BagItem]

    Adds a bag to the bag store.

    Adds a bag to the bag store. A UUID to use as bag-id may be provided. If it is not, it is generated by the function. The bag must be virtually-valid in the context of this bag store, otherwise the function fails. Note that it may be possible to prune the bag before adding it.

    Currently, only unserialized bags are supported.

    By default, a copy of bag will be staged to stagingDir. However, the caller can skip this step and have bag be moved into the bag store from its current location. In that case, the current location must be on the same partition as the bag store itself, so that an atomic move operation is possible.

    A hidden file cannot be added to the bag store, as it would result in a bag that starts off inactive.

    bag

    the bag to add

    optUuid

    the UUID to use as bag-id

    move

    move (instead of copy) the bag into the bag store*

    returns

    the added BagItem
